## Overview

This MSc Dissertation project develops and evaluates multiple machine learning models to enhance **situational awareness and threat detection** in embedded IoT systems. The models are trained to detect real-world cyberattacks — including DDoS, ransomware, and SQL injection — using network traffic data generated by IoT devices, and deployed on edge devices for real-time inference.

## Key Components

### 1. Data Collection & Preprocessing
- Dataset: **ML-EdgeIIoT** — 157,800 instances, 63 features, 15 attack classes, 0 missing values | [Download on Kaggle](https://www.kaggle.com/datasets/mohamedamineferrag/edgeiiotset-cyber-security-dataset-of-iot-iiot) | [IEEE DataPort](https://ieee-dataport.org/open-access/edge-iiotset-new-comprehensive-realistic-cyber-security-dataset-iot-and-iiot-applications)
- Applied data cleaning: dropped irrelevant columns, handled missing values, converted data types
- Ensured dataset consistency and accuracy before modelling

### 2. Feature Selection
- **ANOVA F-test** and **Chi-Squared test** to identify the most significant features
- **VarianceThreshold** to remove constant/low-variance features
- Top 18 features selected: `tcp.flags`, `mqtt.msgtype`, `tcp.ack`, `dns.qry.qu`, `tcp.seq` and others

### 3. Model Development
- Trained and evaluated multiple ML classifiers:
  - Random Forest
  - Support Vector Machine (SVM)
  - K-Nearest Neighbors (KNN)
  - Multi-Layer Perceptron (MLP)
- Hyperparameter tuning via **RandomizedSearchCV**

### 4. Ensemble Learning
- Extended modelling with ensemble techniques:
  - **Bagging**
  - **Boosting (AdaBoost)**
  - **Gradient Boosting**
  - **Stacking**
- Evaluated with accuracy, precision, recall, F1-score, and confusion matrix

### 5. Edge Deployment
- Serialised trained models using **joblib**
- Deployed on edge devices for real-time IoT threat analysis
- Containerised using **Docker** and orchestrated with **Kubernetes**

## Results Summary

| Model | Test Accuracy | Notes |
|-------|--------------|-------|
| **Gradient Boosting** | **92.0%** | Best overall performance |
| **Random Forest** | **91.4%** | Strong F1 across most attack types |
| Boosting (AdaBoost) | **90.9** | Competitive with ensemble peers |
| Bagging | **74.9** | Reduced variance effectively |
| Stacking | **84.4** | Similar to KNN/MLP class-wise |
| MLP | **82.2** | Better than SVM on rare classes |
| KNN | **84.4** | Good multi-class handling |
| SVM | **74.9** | F1 < 0.5 on Password, Uploading, XSS |

**Key Findings:**
- Ensemble methods (Gradient Boosting & Random Forest) achieved **F1-scores above 0.9** for: Backdoor, DDoS-TCP, DDoS-UDP, and Vulnerability Scanner attacks
- All models struggled with rare attack classes: **Fingerprinting** and **MITM** (low recall & F1) due to class imbalance
- SVM showed poor performance on Password, Uploading, and XSS attacks (**F1 < 0.5**)
- Tree-based ensembles consistently outperformed individual classifiers across all metrics

## How to Run

### Prerequisites
- Python 3.8+
- Jupyter Notebook
- Install dependencies:

```bash
pip install scikit-learn pandas numpy matplotlib seaborn joblib
```

### Steps

1. Clone the repository:
```bash
git clone https://github.com/MannyMols/ML_Approaches.git
cd ML_Approaches
```

2. Launch Jupyter Notebook:
```bash
jupyter notebook
```

3. Open `ML_Approaches_IoT-Real.ipynb` and run all cells.
